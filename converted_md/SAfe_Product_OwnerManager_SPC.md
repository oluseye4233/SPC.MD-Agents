### Pillar 4: EXAMPLE PATTERNS

**Example 1: AI Native Feature Story**
```
Title: Intelligent Document Summarization for Legal Contracts

User Story:
As a corporate lawyer, I want the platform to automatically generate concise summaries of 50+ page legal contracts highlighting key terms, obligations, and risks, so that I can review contracts 5x faster while ensuring no critical details are missed.

Acceptance Criteria:
- GIVEN a PDF legal contract up to 100 pages
WHEN uploaded to the system
THEN generate a 1-2 page summary within 30 seconds

- GIVEN the generated summary
WHEN evaluated by legal experts (n=10)
THEN achieve minimum 85% accuracy in identifying key terms

- GIVEN contracts with missing or ambiguous clauses
WHEN summarization occurs
THEN flag uncertainties with confidence scores and request human review

- GIVEN multi-language contracts (English, Spanish, French)
WHEN processed
THEN maintain 80%+ accuracy across all supported languages

AI Requirements:
- Model: GPT-4 or Claude Sonnet with legal domain fine-tuning
- Context: 128K token window for full contract processing
- Prompt Pattern: Chain-of-thought reasoning with clause extraction
- Safety: PII redaction, privilege protection, audit logging
- Performance: 95th percentile latency < 45 seconds

NFRs (Non-Functional Requirements):
- Security: SOC 2 Type II compliance, end-to-end encryption
- Privacy: GDPR/CCPA compliant, no training on customer data
- Scalability: Support 10,000 concurrent summarization requests
- Monitoring: Real-time alerts on accuracy degradation below 80%

Definition of Done:
- Code reviewed and merged to main branch
- Model performance validated on test set (n=500 contracts)
- User acceptance testing completed with 5 beta customers
- Documentation updated (API specs, user guides, runbooks)
- Deployed to production with feature flag enabled
- Metrics dashboard configured for model monitoring
```
