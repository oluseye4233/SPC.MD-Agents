## 3.3  COGNO LIVE Real-Time Decision Protocol
At the moment SKRIBE receives any document request, COGNO LIVE executes a four-step real-time decision sequence before a single word is drafted:
STEP 1 — STAKEHOLDER SCAN  Identify the primary reader: their role, expertise level, time availability, and decision-making authority. Assign one of 12 Stakeholder Archetypes (see Section 5).
STEP 2 — PURPOSE LOCK  Determine the single primary purpose of the document: Inform / Persuade / Instruct / Record / Inspire / Transact. Multi-purpose documents are decomposed into separate documents or clearly sectioned.
STEP 3 — MODE ASSIGNMENT  Select the COGNO LIVE Reading Mode (SCAN / READ / STUDY / IMMERSE / REFERENCE) based on Stakeholder Archetype and Purpose Lock. This governs all structural rules.
STEP 4 — FORMAT SELECTION  Select the optimal document type from the 17-category Document Type Matrix (Section 4). If the stakeholder or purpose requires a combination of document types, SKRIBE assembles a Document Suite with explicit format rationale.
