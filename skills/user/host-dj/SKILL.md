## Primary Workflow

HOST DJ executes across three primary capabilities:

### 1. Hosting Selector Engine (HSE)
Ingest requirements to build a Hosting Requirements Profile (HRP). Evaluate candidate hosts using the 8-Criterion Matrix:
- Stack Compatibility (25%)
- Cost at Target Scale (20%)
- Deployment Simplicity (15%)
- Database Fit (15%)
- CI/CD Integration (10%)
- Compliance Coverage (10%)
- Scalability Ceiling (3%)
- Ecosystem Lock-in Risk (2%)

Select primary and fallback hosts from the 24-platform registry, categorised by Free, Startup, Growth, Enterprise, and Specialized tiers.

### 2. Deployment Journey Generator (DJG)
Generate a phase-gated journey calibrated to one of three complexity tiers:
- **Tier A (Express):** Vercel + Railway + Supabase (12 steps, 2–6 hours).
- **Tier B (Standard):** AWS Amplify, GCP, or DigitalOcean (18 steps, 1–3 days).
- **Tier C (Enterprise):** GKE, EKS, AKS (26 steps, 1–2 weeks).

Formulate numbered phases spanning Pre-Flight setup, Database foundations, Backend build, Frontend binding, and Go-Live validation.

### 3. Seamless Deployment Facilitator (SDF)
Emit essential deployment configurations:
- **Environment Templates:** Normalized `.env.example` configurations mapping server and client keys.
- **CI/CD YAML:** Deployment actions integrating automated testing, build steps, and CLI deployments.
- **Health Checks:** Endpoint implementations pining system database connectivity and uptime.
- **Rollback Protocols:** Numbered recovery procedures (e.g., Railway/Vercel CLI rollbacks) triggered by health failures.
