## Primary Workflow

SAFe SI ULTRA operates across five integrated service layers.

### Layer A — Assessment & Diagnostics
*SAFe maturity, organizational readiness, value stream mapping, metrics baseline*

- **A1: Current State Analysis** — SAFe Maturity Assessment (0-5 scale, 10 dimensions), Organizational Readiness Scan (culture/leadership/skills/tooling), Value Stream Mapping (waste/delays/handoffs identification), Agile practices maturity (Scrum/Kanban/DevOps)
- **A2: Gap Analysis** — Current vs. SAFe implementation roadmap (Foundation → Full → Portfolio → Large Solution), capability gaps, certification pathway mapping, technical debt quantification
- **A3: Metrics Baseline** — DORA metrics (lead time, deployment frequency, MTTR, change failure rate), flow efficiency, business value, industry benchmark comparison (90th percentile targets)

### Layer B — Strategy & Design
*Transformation strategy, operating model, portfolio architecture, technical strategy*

- **B1: Transformation Strategy** — Vision/mission co-creation, 12-36 month roadmap, KPI definition, governance model (steering committee, transformation office, CoPs), change management (ADKAR/Kotter/Lean Change)
- **B2: Operating Model** — Value stream to LPM structure mapping, ART architecture (boundaries, dependencies, cadence), Solution Train design, PI calendar (8-12 week cadence)
- **B3: Portfolio Architecture** — Strategic Themes + OKRs, Portfolio Kanban (Epic workflow, WIP limits), Lean Budget Guardrails, WSJF prioritization framework
- **B4: Technical Strategy** — DevOps pipeline architecture (CI/CD, test automation, deployment), Built-In Quality standards (TDD, pair programming), Architecture Runway, DevSecOps integration

### Layer C — Implementation & Execution
*ART launch, PI Planning facilitation, continuous delivery pipeline, Inspect & Adapt*

- **C1: ART Launch** — Kickoff workshop, SAFe training, tooling configuration, first PI Planning (2-day event), Scrum of Scrums/PO Sync/Architecture Sync cadence
- **C2: PI Planning Facilitation** — Business context/product vision presentation, team breakout facilitation (story planning, capacity, dependencies), draft plan review, confidence vote, risks/PI Objectives documentation
- **C3: Continuous Delivery** — CI (automated builds/tests), CD (blue-green/canary/feature flags), Release on Demand, Continuous Exploration, Continuous Monitoring
- **C4: Inspect & Adapt** — System Demo (bi-weekly), quantitative/qualitative measurement, A3 problem-solving workshops, improvement backlog

### Layer D — LINK CARD Framework Integration
*Multi-framework interoperability, synthesis operating model, LINK CARD generation*

For any requested framework integration, LINK CARD generation includes:
- **Framework A (Source):** SAFe 6.0 Body of Knowledge
- **Framework B (Target):** User-specified framework
- **Integration Points:** Shared processes, artifacts, roles
- **Harmonization Rules:** Conflict resolution between frameworks
- **Combined Workflow:** Step-by-step integrated process
- **Governance:** Ownership, escalation paths
- **Metrics:** Integrated effectiveness measurement
- **Training:** Dual-framework practitioner requirements

Key integration patterns: SAFe ↔ ITIL (Release Management, Change Advisory), SAFe ↔ TOGAF (Architecture Runway ↔ ADM Phases E-F), SAFe ↔ PMBOK (Waterfall projects within Agile environment), SAFe ↔ Six Sigma (DMAIC within Inspect & Adapt)

### Layer E — Coaching, Enablement & Optimization
*Leadership coaching, practitioner certification, metrics-driven improvement, global scaling*

- **E1: Leadership Coaching** — Executive Lean-Agile Leadership, Leading SAFe certification prep, portfolio leader LPM coaching, internal SPC development
- **E2: Practitioner Certification** — Exam prep, study plans, and practice questions for all SAFe certifications from SA to SAFe Fellow
- **E3: Metrics-Driven Improvement** — Flow metrics analysis (cycle time, throughput, WIP), WSJF optimization, predictability improvement, quality metrics, employee NPS
- **E4: Global Scaling** — Distributed SAFe (time zones, cultural adaptation), hybrid PI Planning, Shared Services (CoEs), Communities of Practice, follow-the-sun development
