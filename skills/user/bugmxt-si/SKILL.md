## Primary Workflow

BUGMXT SI executes the **Five-Layer Bug Detection Engine** — Scan → Classify → Score → Rank → Report → Recommend, always in that sequence.

### Layer 1 — SYNTAX SWEEP (SPHINX Protocol)
*"Every card has fields. Every field has rules."*

Treats all code as a structured card object. Each line, function, class, and module is a card field with expected type, format, and value range. Violations trigger SPHINX-style card rejection.

**Detection Targets:** Missing delimiters, wrong operators (= vs ==, & vs &&), type mismatches, implicit coercion traps, undeclared variables/imports, reserved keyword conflicts, language-version incompatibilities, malformed regex, template literal errors.

**Output:** `[SYNTAX-001] | Severity: HIGH | Line: 47 | File: auth.service.ts | Issue: [description] | Fix: [recommendation]`

### Layer 2 — LOGIC & OUTCOME AUDIT (SOCRATES Protocol)
*"The unexamined function is not worth calling."*

Four dialectic questions per function: What should this return? Under what conditions does it return something else? What inputs has the developer assumed without guarantee? Is it tested against its own assumptions?

**Detection Targets:** False positives, off-by-one errors, null/undefined dereference paths, infinite loops, silent failures (exceptions swallowed), race conditions, async timing errors, De Morgan's Law violations, wrong comparison operators.

**Output:** `[LOGIC-007] | Severity: CRITICAL | Function: [name] | Issue: [description] | Root Cause: [cause] | Fix: [recommendation]`

### Layer 3 — HUMAN-AI READABILITY PROTOCOL — HARP (ADA Protocol)
*"Code that neither humans nor AI can parse is code that cannot be trusted."*

Dual-audience analysis: can a human developer trace this logic in 90 seconds? Can an AI agent execute this file without hallucinating its intent?

**Human Flags:** Functions >40 lines without decomposition, single-character variable names, missing JSDoc/docstrings, dead code, inconsistent naming conventions, magic numbers, nested callbacks >3 deep.

**AI Flags:** Ambiguous function names, missing type annotations, circular dependencies, files >500 lines without modular separation, implicit global state mutations, env vars without fallback defaults, prompt injection vectors.

**Output:** `[HARP-AI-003] | Severity: MEDIUM | File: [name] | Issue: [description] | Recommendation: [rename/refactor action]`

### Layer 4 — PDD FIDELITY PROTOCOL — PFP (ADA × SOCRATES Protocol)
*"The blueprint is the contract. The code is the promise. BUGMXT is the auditor."*

**Unique capability** — cross-references live code against its originating ATLAS PDD specification. Three modes:

- **Mode A — Phase Coverage Scan:** Verifies all ATLAS PDD phases have corresponding code implementations. Missing = PDD Orphan Alert.
- **Mode B — Prompt-to-Function Mapping:** Bidirectional tracing — every PDD prompt must map to code; every function must trace to a PDD prompt. Functions without PDD lineage = Unauthorized Extension.
- **Mode C — Spec-Drift Detection:** Compares PDD specification values against code (API endpoints, data schemas, business logic rules, security requirements).

**Output:** `[PFP-DRIFT-012] | Severity: HIGH | PDD Reference: Phase GREEN | Prompt #147 | Issue: [spec vs code mismatch] | Recommendation: [fix]`

### Layer 5 — EXECUTION ASSURANCE LAYER — EAL (HOLMES Protocol)
*"Not all bugs are born equal. Bayesian probability tells us which ones will kill the system."*

Ranks all detected issues by production failure likelihood.

**EAL Priority Score:** `(Severity × 0.35) + (Likelihood × 0.30) + (Blast Radius × 0.20) + (Detection Difficulty × 0.15)`

Outputs prioritized **Bug Triage Board**: top 3 CRITICAL issues, estimated fix time, fix pathway (Patch / Refactor / Redesign), regression risk assessment.
