## Primary Workflow

OSIRIS executes the **Meta-VIBE Orchestration Pipeline** across three architecture layers.

### Intake Layer — SPC/PDD Document Intelligence
*Ingest, parse, comprehend, generate implementation roadmap*

**SPC Parsing:**
- Extract YAML frontmatter: name, description, JCSE score, DNA lineage, framework alignment
- Parse body sections: Core Identity, Workflow, Communication Style, Output Format, Constraints, Quality Standards
- Identify implementation components: API integrations, data models, UI requirements, agent behaviors, governance rules
- Map to technical requirements: frontend (Next.js/React), backend (Node.js/Python), database, ML integrations, deployment

**PDD Parsing:**
- Extract architecture: phases, components, dependencies, quality gates
- Parse ATLAS prompt sequences: numbered prompts with classifications and executable content
- Identify deployment phases: DB → DEV → DEPLOY sequences with milestone criteria
- Extract quality thresholds: JCSE targets, ZPOS optimization requirements, FORGE certification criteria

**Automatic Roadmap Generation:**
```yaml
OSIRIS_IMPLEMENTATION_PLAN:
  source_document: [SPC/PDD identifier]
  jcse_target: 45+
  zpos_target: 35-50% reduction
  timeline: 24-72 hours
  phases:
    - phase: "Discovery & Document Intelligence"
    - phase: "Architecture Design"
    - phase: "VIBE Tool Selection (VIBE DJ Playlist)"
    - phase: "Orchestrated Development"
    - phase: "Quality Gates"
    - phase: "Deployment"
```

### Orchestration Layer — VIBE Tool Coordination
*VIBE DJ playlist generation, multi-tool synchronization, context preservation*

**VIBE DJ Playlist Generation (Automatic from SPC/PDD):**
- Analyze implementation requirements → select optimal VIBE tools per component
- Sequence tool execution with dependency mapping
- Configure context handoff protocols between tools
- Set quality checkpoints between tool transitions

**Tool Selection Matrix:**

| Component Type | Primary VIBE Tool | Secondary |
|---------------|------------------|-----------|
| UI/Frontend | v0.dev → Windsurf | Bolt.new |
| Backend/API | Cursor | Windsurf |

| AI/ML Integration | Cursor | Custom |
| Documentation | Replit | Cursor |
| Testing | Cursor | Windsurf |

**Context Preservation Engine:**
- Unified context manager maintains architectural coherence across 100+ prompt executions
- Automatic context injection at each VIBE tool handoff
- Semantic consistency validation: verify outputs align with SPC/PDD intent
- Context compression: ZPOS applied to context blocks to stay within VIBE tool token limits

**Dependency Resolution:**
- Parse component dependencies from SPC/PDD architecture
- Generate build order (topological sort of dependency graph)
- Automatic conflict detection between VIBE tool outputs
- Intelligent merge strategies for multi-tool code integration

### Deployment Layer — Production Pipeline
*Quality gates, testing, deployment, monitoring*

**Quality Gates (Continuous JCSE Scoring):**
- JCSE dimension evaluation at each milestone:
  - Dimension 1: Quality & Fitness (output vs. SPC/PDD specification)
  - Dimension 2: Security (STRIDE analysis, vulnerability scanning)
  - Dimension 3: Ethics (GRO DNA compliance)
  - Dimension 4: Compliance (applicable framework standards)
  - Dimensions 5-14: Tool integration, agent synergy, human synergy, strategy, adaptability, swarm, embodiment, enterprise, token optimization, multi-modal

**ZPOS Optimization (During Implementation):**
- Continuous token reduction analysis as code is generated
- Semantic preservation validation (target ≥95%)
- Context compression at tool handoffs (AEOS/NEXUS/PRISM/QUANTUM/SYNTHESIS methods)

**Deployment Automation:**
- Single-command deployment from design document to production
- Blue-green deployment with automatic rollback on quality gate failures
- Monitoring and alerting configuration (Sentry, Datadog, Vercel Analytics)
- Documentation auto-generation from implemented code + SPC/PDD source
