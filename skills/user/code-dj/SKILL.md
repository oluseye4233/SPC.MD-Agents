## Primary Workflow

CODE DJ executes the **7-Phase PromptWare-to-Codebase Conversion Engine (PCE)** — always in sequence: Understand → Allocate → Justify → Generate → Validate → Compress → Deliver.

### Phase 1 — Document Ingestion & Classification
Accepts: ATLAS PDD, SDD, BRD, Project Charter, Architecture Diagrams (Mermaid/PlantUML). Classifies type, assesses completeness, assigns PCE confidence (HIGH/MEDIUM/LOW). Confidence < 70% triggers SOCRATES interrogation before proceeding.

**Output:** Document Classification Report with completeness score, ambiguity flags, and proceed decision.

### Phase 2 — ATLAS Parsing & Atomic Decomposition
Decomposes every prompt, feature, or user story into tagged atomic implementation units:
- Implementation Type (API | UI component | schema | AI prompt | config | infrastructure | test)
- Target Layer (Frontend | Middleware | Backend | Data | AI/ML | Infrastructure | Security)
- Target Platform (Web | Mobile | Desktop | Server | Embedded | Robot | GPU)
- Language Candidates (primary + fallback) and upstream/downstream dependencies

**Output:** PCE Atomic Implementation Map — directed acyclic graph (DAG) of all implementation units.

### Phase 3 — Code Combination Advisory (VIBE DJ × HOLMES)
Project-specific, document-derived technology stack selection accounting for environment constraints, team profile, performance requirements, compliance needs (GDPR/HIPAA/SOX), budget signals, and scalability targets.

**Output:** Optimal Development Blueprint — complete technology stack with platform-by-platform allocation and evidence-based justification matrix.

### Phase 4 — Codebase Architecture Design (ADA ULTRA SI × FRACTAL)
- Directory structure and file organization per target platform
- Module boundaries and interface contracts
- Data flow diagrams (Frontend → API → Business Logic → Data)
- API contract specifications (REST/GraphQL/gRPC)
- Database schema derived from PDD data entities
- Authentication/authorization architecture + testing strategy per layer

**Output:** Codebase Architecture Document — complete structural specification.

### Phase 5 — Code Generation (Full-Stack AI Architect × DEXTER)
Generates complete, runnable code in dependency order for each atomic unit:
- Full code in the selected language (not pseudocode or stubs)
- Inline documentation (docstrings, JSDoc, comments)
- Type annotations (TypeScript types, Python hints, Go interfaces)
- Unit test scaffolding per function/method
- Configuration files, env templates, deployment manifests

**Implementation Environments:**

| Environment | Optimized Output |
|-------------|-----------------|
| GitHub | Full repo: README, .gitignore, CI/CD workflows, branch strategy, PR templates |
| Jupyter | Modular cells with markdown, executable blocks, visualization hooks |
| Replit | Single-workspace structure with run commands and package management |
| Cursor / Windsurf | Project structure with AGENTS.md configuration |
| Bolt.new | Full-stack template with preview and deployment hooks |
| Claude Code | AGENTS.md-governed repo with SDD/SPC configuration |

### Phase 6 — PDD Fidelity Validation (BUGMXT × SOCRATES)
- **Coverage Check:** Every PDD prompt ID maps to at least one code file or function
- **Orphan Detection:** Every code file traces to at least one PDD prompt — zero unauthorized code
- **Contract Validation:** Every API endpoint matches PDD interface specification
- **Logic Audit:** SOCRATES interrogates every complex algorithm against stated requirements
- **Security Scan:** ARES reviews all generated code for injection vectors, auth flaws, secret leakage

**Output:** PCE Fidelity Report — pass/fail matrix against every PDD prompt with remediation for gaps.

### Phase 7 — SPARTAN Compression & Delivery (SPARTAN × ZPOS)
Removes dead code, merges redundant modules, collapses infrastructure to MVP footprint, applies ZPOS+5 to documentation. Delivers:
- Compressed production-ready codebase (60–85% file reduction)
- Code-to-PDD Traceability Map (every function traces to its originating prompt)
- Upgrade Path (CLASS C deferred features with re-integration instructions)
- Deployment Guide for selected implementation environment
